# WorldTree Publishing System: Requirements & Technical Approach

## Executive Summary

The current WorldTree content publishing mechanism in the Druids system lacks deterministic behavior and machine-readable output structures. This document outlines requirements and a technical approach for implementing a robust, RAG-ready publishing system that enables agents to leverage previous session outputs in future collaborations.

## Problem Statement

### Current Issues
1. **Non-deterministic Publishing**: Content publishing depends on session detection rather than explicit configuration
2. **Inconsistent Storage**: Mixed approaches between filesystem and WorldTree namespaces
3. **Poor Machine Readability**: Content stored without semantic structure for RAG retrieval
4. **Missing Agent Access**: No clear mechanism for agents to discover and leverage published content

### Impact
- Agents cannot build upon previous work effectively
- Coordination sessions operate in isolation without institutional memory
- Knowledge accumulation potential is lost
- Evolution and learning capabilities are severely limited

## Requirements

### Functional Requirements

#### FR-001: Deterministic Publishing Configuration
- Each coordination session MUST explicitly define output publishing intent
- Session descriptions MUST specify WorldTree namespace targets
- Publishing behavior MUST be predictable and configurable per session type

#### FR-002: Machine-Readable Content Structure
- Published content MUST include semantic metadata for RAG retrieval
- Content MUST be structured with clear topic, type, and context classifications
- Embeddings or search indices MUST be generated for content discovery

#### FR-003: Agent Content Discovery
- Agents MUST be able to query WorldTree for relevant prior work
- Search capabilities MUST support semantic similarity and topic-based retrieval
- Content access MUST respect namespace permissions and agent capabilities

#### FR-004: Progressive Knowledge Accumulation
- New sessions MUST be able to reference and build upon previous outputs
- Knowledge graphs MUST be updated with new content relationships
- Content versioning MUST track evolution and improvement chains

### Technical Requirements

#### TR-001: Unified Storage Architecture
- Single authoritative WorldTree storage backend
- Consistent namespace organization across all content types
- Performance optimized for both write and query operations

#### TR-002: RAG Integration Points
- Content automatically indexed for vector similarity search
- Metadata structured for filtered retrieval (by topic, date, participants, etc.)
- Integration hooks for LLM context injection during agent interactions

#### TR-003: Session Configuration Schema
- Declarative publishing configuration in session requests
- Support for multiple output targets per session
- Template system for common publishing patterns

    agentId: string,

    ragContext?: RAGContext
  ): Promise<FilteredContentResults> {
    
    // Get agent's realm access permissions
    const agent = await this.getAgent(agentId);

    const allowedNamespaces = this.buildAllowedNamespaces(realmAccess);
    
    // Query with domain boundaries enforced
    const filteredQuery = {
      ...query,
      namespaceFilter: allowedNamespaces,
      domainTags: realmAccess.domains,
      crossRealmAccess: realmAccess.allowCrossRealm
    };
    
    const results = await this.worldTreeService.query(filteredQuery);
    
    // Apply additional domain-specific filtering
    return this.applyDomainFiltering(results, realmAccess);
  }
  
  private buildAllowedNamespaces(realmAccess: RealmAccess): string[] {
    const namespaces = [];
    
    // Agent's bound realm (for Elementals)
    if (realmAccess.boundRealmId) {
      namespaces.push(`worldtree://realm-scoped/${realmAccess.boundRealmId}/*`);
    }
    
    // Agent's accessible realms (for Druids)
    for (const realm of realmAccess.accessibleRealms || []) {
      namespaces.push(`worldtree://realm-scoped/${realm.realmId}/*`);
    }
    
    // Shared cross-realm content (if permitted)
    if (realmAccess.allowCrossRealm) {
      namespaces.push('worldtree://public/cross-realm-shared/*');
    }
    
    return namespaces;
  }
